# Kotoba Note 详细功能与技术架构报告

## 1. 项目概述

Kotoba Note 是一款定位为“更贴近真实学习节奏”的日语单词手帐应用。作为一款深度结合数字化便捷性与手帐式人文感的产品，它旨在打破机械化背诵的枯燥感。通过技术驱动的流程优化，应用将碎片化的记单词行为转化为一个更具“有人味”、顺手且利于长期坚持的学习闭环。其核心愿景是利用自动化手段辅助用户进行文本分析，同时保留学习过程中的自主思考空间。

## 2. 核心技术栈分析

本项目采用 `100% Python` 语言环境构建，基于 `Local-first（本地优先）` 的架构理念，确保了用户数据的私密性与极快的响应速度。

| 技术组件 | 用途说明 | 预期作用 |
| --- | --- | --- |
| PySide6 | 跨平台 GUI 框架（Qt for Python） | 实现 UI 与业务逻辑的解耦，提供工业级稳定的桌面交互体验与质感 |
| SQLite | 轻量级关系型数据库 | 建立本地数据持久化层，支撑词库管理、学习进度及历史记录的高效检索 |
| Janome | 日语形态学分析引擎 | 执行分词（Tokenization）与词性标注（POS Tagging），实现自动化文本解析 |

## 3. 功能模块深度解析

### 3.1 课文手札

该模块是应用的核心入口。系统支持导入原生日语文本，利用 Janome 的形态学分析能力，自动提取重点词汇并统计词频。同时，通过分析结果自动为汉字添加注音（Furigana），极大地提升了用户处理长篇课文素材的效率。

### 3.2 词汇本管理

提供了精细化的数据管理能力。用户可根据学习状态进行多维筛选，支持标签化分类、优先级设定及批量整理。内置的回收站机制提供了数据回滚保障，确保误删的词条可以随时恢复。

### 3.3 复习计划系统

基于学习节奏算法动态编排每日复习任务。该系统支持“汉译日”与“日译汉”等多个维度的交叉练习，以强化用户对词汇的深度记忆。

### 3.4 随机自测功能

提供即时的答题反馈。在交互设计上，应用特意保留了“学习留白”机制，允许用户在答题完成后停留回看，避免陷入单纯追求效率的“刷题陷阱”，鼓励深度思考。

### 3.5 错题追踪（错题本）

系统自动追踪高频失误词汇，生成专项练习。其核心逻辑在于支持错题在掌握后重新“回流”至主复习库，形成一套闭环的动态纠错机制。

### 3.6 学习报告

通过可视化手段呈现学习趋势、词汇分布及进度数据。报告不仅记录过去的行为，更会基于数据分布提供下一步的学习建议，辅助用户科学调整策略。

## 4. 项目结构与代码组织

项目采用清晰的模块化设计，代码组织结构如下：

- `app/`：核心应用逻辑目录，包含 GUI 界面组件、业务逻辑处理及数据库交互模块
- `tests/`：包含针对核心功能的冒烟测试（Smoke Tests）及相关单元测试用例，确保系统迭代的稳定性
- `.gitignore`：版本控制配置文件，规定了忽略提交的临时文件及环境文件
- `README.md`：项目说明文档，包含项目简介、功能概述及开发指南
- `main.py`：程序入口文件，负责初始化应用上下文并启动主事件循环
- `requirements.txt`：项目依赖清单，明确了 `PySide6`、`Janome` 等运行环境版本

## 5. 开发阶段与项目现状总结

Kotoba Note 目前已完成 `V1.0` 核心主流程的开发，具备完整可运行的“分析 - 收集 - 复习 - 测验 - 反馈”链路。

### 当前优化方向

项目现处于“精细化打磨”阶段。后续重点将集中在以下几个方向：

- 学习逻辑的深度调优，例如更细的记忆曲线与题目调度策略
- 反馈节奏的科学化控制，让练习既不打断用户，也不让人茫然
- 界面视觉质感的持续提升，进一步增强产品辨识度
- 更多体现“有人味”的人性化交互细节，降低学习疲劳感

目标是把它继续打磨成一个温和、高效、适合长期使用的日语学习生态工具。
